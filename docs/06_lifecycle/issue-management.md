## GitHub Projectsカンバン列

| 列 | 説明 |
|----|------|
| Backlog | 未着手・優先度未確定 |
| Todo | 着手可能（優先度確定済み） |
| In Progress | 作業中 |
| Review | PR作成済み・レビュー待ち |
| Done | 完了（テスト通過 + レビュー承認 + stg動作確認） |

## Done定義

以下の全てを満たした場合にDoneとする:
- [ ] 実装完了
- [ ] テスト通過（Unit + Integration）
- [ ] コードレビュー承認
- [ ] ステージング環境での動作確認
- [ ] ドキュメント更新（必要な場合）

## Claude CodeがIssueを取得して作業する際のルール

1. Issueの内容を確認し、不明点があればコメントで確認を求める
2. ブランチを作成: `feature/[issue番号]-[概要]` or `fix/[issue番号]-[概要]`
3. CLAUDE.mdのルールに従って実装
4. テストを作成・実行
5. PRを作成し `Closes #[issue番号]` で紐づけ
6. **勝手に要件を解釈しない** — 不明な場合は必ず確認

## Issueテンプレート

```markdown

<!-- 何を実現したいか -->

## 背景・動機
<!-- なぜ必要か -->

- [ ] 基準2

## デザイン参照
<!-- 関連するワイヤーフレーム・デザイン画像のパス -->

```

