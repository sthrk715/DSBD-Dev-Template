# バグ修正レシピ

## 手順

### Step 1: 情報収集
1. GitHub Issueのバグ報告を確認（再現手順、スクリーンショット）
2. 関連するログ・エラーメッセージを確認
3. 影響範囲を特定

### Step 2: 再現確認
1. ローカル環境でバグを再現
2. 再現手順を明確にする
3. 関連するコードを特定

### Step 3: 原因調査
1. git log / git blame で変更履歴を追跡
2. 関連するテストの有無を確認
3. 根本原因（Root Cause）を特定

### Step 4: 修正
1. まずバグを再現するテストを書く（Red）
2. 修正を実装（Green）
3. リグレッションテストを追加
4. 既存テストが全てパスすることを確認

### Step 5: ブランチ・PR
```
git checkout -b fix/[issue番号]-[バグ概要]
```

## チェックリスト
- [ ] バグの再現確認
- [ ] 根本原因の特定
- [ ] 再現テスト作成
- [ ] 修正実装
- [ ] リグレッションテスト追加
- [ ] 全テスト通過確認
- [ ] PR作成（Closes #XX）
