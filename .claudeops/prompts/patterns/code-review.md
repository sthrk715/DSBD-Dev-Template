# コードレビューパターン

## 使用方法
PRレビューやファイル単位のコードレビューに使用する。

## プロンプト

以下のコードをレビューしてください。

レビュー観点（優先順）:
1. **セキュリティ**: OWASP Top 10の観点。SQLi, XSS, CSRF, 認証認可の不備
2. **正確性**: ビジネスロジックの正しさ、エッジケースの処理
3. **保守性**: コードの可読性、命名の適切さ、関数の長さ（30行以内）
4. **パフォーマンス**: N+1クエリ、不必要な再レンダリング、大量データ処理
5. **テスタビリティ**: テストしやすい設計になっているか

AI生成コード固有の確認:
- 存在しないライブラリ/APIの参照がないか
- TODO/FIXMEプレースホルダーが放置されていないか
- 実装が要件に対して過剰でないか

対象: [ファイルパスまたはPR番号]

## 出力形式
| 重要度 | ファイル:行 | 指摘内容 | 改善案 |
