# テスト生成パターン

## 使用方法
このプロンプトをClaude Codeセッションで使用し、対象ファイルのテストを生成する。

## プロンプト

以下のファイルに対してユニットテストを生成してください。

要件:
- テストフレームワーク: Vitest（CLAUDE.mdの規約に従う）
- カバレッジ目標: 行カバレッジ80%以上
- テスト構造: Arrange-Act-Assert パターン
- 以下のケースを必ず含む:
  - 正常系（happy path）
  - 境界値
  - エラーハンドリング
  - エッジケース（null, undefined, 空配列, 空文字列等）
- モックは最小限にし、テストの意図が明確になるようにする
- テスト名は日本語で「〜の場合、〜となること」形式
- `__tests__/` ディレクトリに配置

対象ファイル: [ファイルパス]

## 注意事項
- 既存テストがある場合は整合性を確認すること
- 外部依存（BigQuery, Cloud SQL等）はモックで分離すること
- テストのトートロジー（実装をそのまま複製したテスト）を避けること
- データベーステストにはテスト用フィクスチャを使用すること
